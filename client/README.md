# Seller Desk

СПА-приложение для тестового задания: кабинет продавца с просмотром, фильтрацией и редактированием объявлений, включая AI-подсказки для описания и цены.

## Стек

- React
- TypeScript
- Vite
- React Router
- Axios
- Zustand
- Ant Design
- Day.js

## Что реализовано

- Список объявлений `/ads`
  - поиск
  - фильтрация по категориям
  - фильтр `только требующие доработок`
  - сортировка
  - пагинация по 10 элементов
  - loading / error / empty states
- Просмотр объявления `/ads/:id`
  - полная карточка
  - блок с незаполненными полями
  - переход к редактированию
- Редактирование `/ads/:id/edit`
  - форма на Ant Design
  - динамические поля `params` по `category`
  - сохранение через `PUT /items/:id`
  - автосохранение черновика в `localStorage`
  - восстановление или сброс найденного черновика
  - базовая защита от потери несохранённых изменений
- AI service layer
  - прямой вызов локального Ollama
  - структурированные ответы для описания и цены

## Запуск

### 1. Backend

```bash
cd server
npm i
npm start
```

По умолчанию сервер поднимается на `http://localhost:8080`.

### 2. Frontend

```bash
cd client
npm i
cp .env.example .env
npm run dev
```

Фронтенд ожидает API по `VITE_API_BASE_URL`.

## Настройка AI

Для локального Ollama:

```bash
ollama pull llama3
ollama serve
```

И в `.env`:

```env
VITE_OLLAMA_URL=http://localhost:11434/api/generate
```

## Принятые решения

- Zustand используется только для состояния списка: поиск, фильтры, сортировка, пагинация.
- Состояние формы редактирования хранится локально в Ant Design Form.
- Для AI не используется внешний секретный API с клиента. Используется локальный Ollama.
