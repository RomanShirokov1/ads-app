# AI-ассистент для улучшения объявлений

Тестовое задание стажёра Frontend (весенняя волна 2026): личный кабинет продавца с просмотром, фильтрацией и редактированием объявлений, плюс AI-помощник для текста и цены.

## Технологии

- Frontend: React, TypeScript, Vite, React Router, Zustand, Ant Design, Axios
- Тесты: Vitest
- Качество кода: ESLint, Prettier

## Что реализовано

### Обязательная часть

- Страница `/ads`
  - поиск по названию
  - фильтры по категориям
  - фильтр «только требующие доработки»
  - сортировка
  - пагинация по 10 объявлений
  - состояния загрузки/ошибки/пустого результата
- Страница `/ads/:id`
  - полная карточка объявления
  - блок «Требуются доработки» с незаполненными полями
  - переход к списку и к редактированию
- Страница `/ads/:id/edit`
  - редактирование категории, названия, цены, характеристик и описания
  - сохранение через `PUT /items/:id`
  - сохранение черновика в `localStorage`
  - восстановление/сброс черновика
  - защита от ухода со страницы при несохранённых изменениях
- AI-функции
  - «Придумать/Улучшить описание»
  - «Узнать рыночную цену»
  - применение результата AI в форму

### Дополнительные функции (со звёздочкой)

- отмена запросов через `AbortController` при смене фильтров
- unit-тесты (Vitest)
- переключение светлой/тёмной темы с сохранением в `localStorage`

## Требования

- Node.js 20+
- npm 10+

## Переменные окружения

Создайте файл `client/.env`

```env
VITE_API_BASE_URL=http://localhost:8080
VITE_OLLAMA_URL=http://localhost:11434/api/generate
```

## Запуск

### 1. Backend

```bash
cd server
npm install
npm start
```

Сервер по умолчанию: `http://localhost:8080`

### 2. Frontend

```bash
cd client
npm install
npm run dev
```

Клиент по умолчанию: `http://localhost:5173`

## Настройка LLM

### Вариант 1 (использован в проекте): локально через Ollama

1. Установить Ollama: <https://ollama.com>
2. Загрузить модель:

```bash
ollama pull llama3
```

3. Запустить сервер Ollama:

```bash
ollama serve
```

4. Проверить `VITE_OLLAMA_URL` в `client/.env`:

```env
VITE_OLLAMA_URL=http://localhost:11434/api/generate
```

## Принятые решения

- Разделение состояния:
  - список объявлений (фильтры/пагинация) хранится в Zustand
  - форма редактирования управляется Ant Design `Form`
- Черновик:
  - автосохранение с debounce для снижения частоты записи в `localStorage`
  - безопасная запись в storage с обработкой ошибок `quota/security`
- UI-обёртки:
  - вынесены общие контролы для Ant Design (`UiButton`, `UiInput`, `UiSelect` и т.д.)
  - общие классы для dropdown/popover вынесены в shared overlays
- Минимальные правки backend для локальной работы:
  - CORS для frontend
  - поддержка `PORT`
  - корректная выдача `id` в списке объявлений
